# 失败模式库（第一版）

## 1. 启动脚本漂移
### 症状
- `start.sh` 指向不存在的 `app.py`
- 实际入口已经改成 `main.py`

### 根因
- 代码进化了，启动脚本没进化
- 缺少自动检查

### 修复
- 收敛标准入口
- 加文件存在性检查

---

## 2. 虚拟环境命名不一致
### 症状
- 文档写 `venv/`
- 实际目录是 `.venv/`

### 根因
- 人工习惯漂移
- 没有“单一真相源”

### 修复
- 明确约定只认一个目录名
- 脚本、README、AGENTS 同步

---

## 3. 文档与真实入口脱节
### 症状
- README 仍在教旧命令
- 新维护者/AI 第一次就跑错

### 根因
- 文档没有被当成代码管理
- 没有一致性检查

### 修复
- README 与脚本绑定检查
- 改入口必须同时改文档

---

## 4. Tunnel 逻辑分叉
### 症状
- 多个 tunnel 脚本，各自代理不同端口/路径
- 不知道哪个才是正式入口

### 根因
- 历史试验没有收敛
- 公网暴露没有标准化

### 修复
- 只保留一个标准 tunnel 脚本
- 历史脚本转发到标准入口

---

## 5. 服务“启动成功”但功能不可用
### 症状
- 进程在，端口开了
- 首页/关键 API 实际访问失败

### 根因
- 缺少 smoke test
- 只验证了进程，没有验证能力

### 修复
- 建最小功能烟雾测试
- 把 smoke test 纳入交付标准

---

## 6. 思想先进，脚本落后
### 症状
- 方法论文档很漂亮
- 实际项目依然靠手工和记忆维持

### 根因
- 只升级 SKILL.md，没有升级模板、脚本和 playbook

### 修复
- 把生产经验沉淀成模板与脚本
- 让 Skill 具备“可复制落地”的能力
