# 天工引擎 · Harness Engineering

> 把 AI 从「不可控的魔法」变成「可工程化的生产要素」。

[![Version](https://img.shields.io/badge/version-1.0.0-blue)](https://github.com/chengbenchao/harness-engineering)
[![License](https://img.shields.io/badge/license-MIT-green)](LICENSE)

## 🎯 核心理念

**人类掌舵，智能体执行。**

构建软件仍然需要纪律，但纪律更多地体现在支撑结构上，而不是代码上。保持代码库一致性的工具、抽象和反馈回路变得越发重要。

## 🏗️ 四层架构模型

| 层级 | 解决什么问题 | 关键机制 |
|------|-------------|----------|
| **Prompt** | 单次交互质量 | 清晰表达需求，减少误解 |
| **Context** | 系统性协作 | 项目上下文、标杆代码、历史决策 |
| **Agent** | 专业化分工 | 多专家角色、互相审查、责任边界 |
| **Harness** | 完整操作系统 | Hook 拦截、验证闭环、知识进化 |
